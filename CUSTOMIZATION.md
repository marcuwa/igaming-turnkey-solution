# Customization

The platform can be customized based on operator requirements, brand identity, target market, payment model, and game provider setup.

---

## 1. UI/UX Customization

Available UI customization options:

- Logo
- Brand colors
- Homepage layout
- Casino lobby layout
- Game cards
- Navigation menu
- Footer content
- Player wallet page
- Promotion page
- Login/register page
- Mobile layout
- Dark/light theme
- Language/localization

Recommended client inputs:

- Brand name
- Logo file
- Preferred colors
- Reference websites
- Target language
- Target market
- Required pages
- Promotion style

## 3. Agent System Customization

Agent system customization options:

- Number of agent levels
- Operator-to-agent transfer rules
- Agent-to-sub-agent transfer rules
- Agent-to-player transfer rules
- Balance recovery rules
- Transfer limits
- Agent commission logic
- Settlement reporting
- Agent visibility permissions
- Player assignment rules

Example structures:

```text
Simple:
Operator → Agent → Player

Advanced:
Operator → Master Agent → Agent → Sub-Agent → Player

Private Network:
Operator → Manager → Player Group
```

## 8. Custom Development Process

Typical customization workflow:

1. Client shares requirements.
2. We review platform fit and required changes.
3. We define customization scope.
4. UI/backend changes are implemented.
5. Demo environment is prepared.
6. Client reviews the result.
7. Final adjustments are applied.
8. Deployment or handover is completed.

---

## 9. What to Prepare Before Contacting

To speed up discussion, prepare:

- Target market
- Preferred payment model
- Required game categories
- Required providers, if any
- Agent hierarchy requirement
- UI reference examples
- Required languages
- Expected launch timeline
- Deployment preference
